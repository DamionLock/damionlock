Go live

Putting damionlock.com online

No command line needed — we'll upload through GitHub's website by dragging files, then flip on free hosting.

About 10 minutes · do it once
Part A · Create the repository
1

Unzip the download

Find the file you just downloaded and unzip it (double-click it on Mac, or right-click → Extract on Windows). You'll get a folder full of the site's files.

2

Create a new repository on GitHub

Go to github.com/new. Set Repository name to damionlock, leave it Public, and click Create repository. Don't tick any of the "add a README" boxes.

Part B · Upload the files
3

Open the upload page

On your new empty repo, click the link that says “uploading an existing file” (it's in the grey box of instructions).

4

Drag in everything from the folder

Open the unzipped folder, select all the files and folders inside it (Ctrl+A / Cmd+A), and drag them onto the GitHub upload page.

Drag the contents of the folder, not the folder itself. You can delete connect-github-guide.html, upload-guide.html and the uploads folder first — they're just my helper files and aren't needed on the live site.
5

Commit

Scroll down and click the green Commit changes button. Wait for the files to finish uploading.

Part C · Turn on hosting
6

Enable GitHub Pages

In the repo, go to Settings → Pages. Under Build and deployment → Source, choose GitHub Actions. The deploy pipeline I included does the rest automatically.

7

Point your domain at GitHub

In your domain provider's DNS settings for damionlock.com, add these records:

TypeHostValue
A@185.199.108.153
A@185.199.109.153
A@185.199.110.153
A@185.199.111.153
CNAMEwwwdamionlock.github.io
Replace damionlock in the CNAME value with your actual GitHub username if it's different.
8

Set the custom domain

Back in Settings → Pages, type damionlock.com into the Custom domain box and save. Once it's ready, tick Enforce HTTPS. DNS can take a few hours to fully kick in.

That's it — you're live. 🌱

From now on, updating the site is just: edit a file in the repo (or upload a new one) → it auto-deploys in about a minute. To blog, add a Markdown file in posts/ — full instructions are in README.md.
